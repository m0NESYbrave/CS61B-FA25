BucketsSpeed: We have to iterate through the corresponding bucket when we insert an item.
For List, we can access contiguous memory so overhead is small.
For HashSet, memory is usually not contiguous. Even worse, we don't make use of HashSet's
amortized constant time operations since bucket is type of Collection<Node>, so we can only
iterate through the bucket to search for the key instead of using HashSet's own operations.
If we were able to use a constant-time search over HashSet, we can get a speed-up asymptotically.
This tells us that the selection of the data structure must match with the way we use it.
